# 公司网球积分排行软件 · 设计定稿 v2

> **v2 变更**：新增**单打**——单打与双打各自**独立积分与独立排行榜**（同一人有两套 Elo）；明确**实施阶段**，第一阶段（球员录入 + 比赛录入 + 积分排行，手机网页版）已落地。
> 其余规则不变：仅打一盘（无三盘两胜）、共 4 种赛制、K 值统一 24、积分严格零和、四局短盘 4/5 局封顶均允许。

---

## 1. 产品需求说明

### 1.1 定位
公司内部娱乐用的网球积分系统，**同时支持双打与单打**。双打搭档经常变化，因此积分是**个人 Elo 积分**，不是固定组合积分；单打即标准 1v1。规则要求公平、简单、可解释，**胜负为主、比分仅轻微修正**。

单打与双打是不同能力，因此**两种模式各算各的积分、各排各的榜**，互不影响。

### 1.2 核心原则（实现时必须守住）
- 积分是个人积分，搭档变化不影响积分归属。
- 单打、双打积分相互独立，分别从 1500 起算，各自一套排行榜。
- 胜负是主因，比分系数只做小幅修正（抢 7 / 抢 11 的影响上限不超过标准一盘）。
- 所有人都进排行榜，**不设最低场次门槛**；但必须显示场次和状态标签，避免「打 1 场就稳居第一」的误读。
- 积分**严格零和**：每场赢方所得之和 = 输方所失之和（单打 1+1，双打 2+2）。

### 1.3 功能清单与实施阶段

|---|---|---|
| 1 | **球员录入**：新增、删除（无记录者） | **第一阶段（已实现）** |

| 3 | **积分排行**：单/双打切换、全员展示、多级排序、最近 5 场、天气与时间动态球场背景 | **第一阶段（已实现）** |

## 2. 积分规则说明

### 2.1 基础
- 初始积分：**1500**（单打、双打各一套）
- K 值：**统一 24**（所有人、所有场次、两种模式）。新人快慢通过状态标签体现，不用 K 区分——这是严格零和的前提。

### 2.2 计算流程（单打、双打通用）
一场比赛分两队，每队 **1 人（单打）或 2 人（双打）**，两端人数一致。

```
队均分:  R_A = 队 A 成员积分平均   （单打时即该选手本人分）
         R_B = 队 B 成员积分平均

理论胜率: E_A = 1 / (1 + 10^((R_B - R_A) / 400))
         E_B = 1 - E_A

每人积分变化:
  Δ = K × (S - E_team) × 赛制权重 × 比分系数

  S = 1（胜方） / 0（负方）
  E_team = E_A（A 队选手） / E_B（B 队选手）
  一场内所有选手共用同一个 K、权重、比分系数
```

**零和验证**：胜方每人 `+f·(1-E_A)`，负方每人 `-f·(1-E_A)`（`f = K·权重·系数`）。单打两人一加一减，双打两加两减，总和恒为 0。✅

### 2.3 赛制权重（两种模式共用）

| 赛制 | match_type | 权重 |
|---|---|---|
| 抢 7 | `tiebreak7` | 0.45 |
| 抢 11 | `tiebreak11` | 0.60 |
| 四局短盘 | `short4` | 0.80 |
| 标准一盘 | `standard_set` | 1.00（基准，最高） |

### 2.4 比分修正系数

**抢 7**（按分差，胜方需净胜 2 分）

| 分差 | 例 | 系数 |
|---|---|---|
| 2 | 7:5 / 8:6 / 9:7 | 0.90 |
| 3 | 7:4 | 1.00 |
| 4–5 | 7:3 / 7:2 | 1.10 |
| ≥6 | 7:1 / 7:0 | 1.20 |

**抢 11**（按分差）

| 分差 | 例 | 系数 |
|---|---|---|
| 2 | 11:9 / 12:10 | 0.90 |
| 3–4 | 11:8 / 11:7 | 1.00 |
| 5–6 | 11:6 / 11:5 | 1.10 |
| ≥7 | 11:4 / 11:3 | 1.20 |

**四局短盘**（4/5 局封顶通用，按局差）

| 局差 | 例 | 系数 |
|---|---|---|
| 1 | 4:3 / 5:4 | 0.90 |
| 2 | 4:2 / 5:3 | 1.00 |
| 3 | 4:1 | 1.12 |
| ≥4 | 4:0 | 1.20 |

**标准一盘**（按具体比分查表，未列比分按局差兜底）

| 比分 | 系数 |
|---|---|
| 7:6 | 0.85 |
| 7:5 | 0.95 |
| 6:4 | 1.00 |
| 6:3 | 1.08 |
| 6:2 | 1.15 |
| 6:1 | 1.20 |
| 6:0 | 1.25 |

### 2.5 状态标签（按模式各自统计，单标签取一个）

| 条件 | 标签 |
|---|---|
| 该模式最近 60 天没打（优先级最高，覆盖其他） | 非活跃 |
| 该模式总场次 ≥10 | 稳定 |
| 4–9 | 观察中 |
| 1–3 | 样本少 |
| 0 | 新登记（不出现在该模式榜上） |

### 2.6 排行榜排序（每个模式各自排，多级）
1. 当前 Elo 积分（降序）
2. 胜率（降序，0 场记为 0）
3. 场均归一化净优势（降序）= 各场 `(己方分−对方分)/(己方分+对方分)` 的平均
4. 近 10 场净胜（W − L，降序）

> Elo 为第一排序，后三级基本只在并列时才决定先后。

## 3. 数据库表结构（SQLite）

> 设计要点：**真值是 `players` + `matches`**；各模式的当前积分、历史记录都由「按时间重放比赛」推导。`player_ratings` 和 `rating_history` 是重算产物（缓存），任何录入/删除后重建。

```sql
-- 球员身份
CREATE TABLE players (
    id         INTEGER PRIMARY KEY AUTOINCREMENT,
    name       TEXT NOT NULL UNIQUE,
    created_at TEXT NOT NULL DEFAULT (datetime('now')),
    updated_at TEXT NOT NULL DEFAULT (datetime('now'))
);

-- 比赛（单打时 *_player_2_id 为 NULL）
CREATE TABLE matches (
    id                 INTEGER PRIMARY KEY AUTOINCREMENT,
    date               TEXT    NOT NULL,            -- YYYY-MM-DD
    mode               TEXT    NOT NULL,            -- 'singles' / 'doubles'
    match_type         TEXT    NOT NULL,            -- tiebreak7/tiebreak11/short4/standard_set
    team_a_player_1_id INTEGER NOT NULL REFERENCES players(id),
    team_a_player_2_id INTEGER     NULL REFERENCES players(id),
    team_b_player_1_id INTEGER NOT NULL REFERENCES players(id),
    team_b_player_2_id INTEGER     NULL REFERENCES players(id),
    team_a_score       INTEGER NOT NULL,
    team_b_score       INTEGER NOT NULL,
    winner_team        TEXT    NOT NULL,            -- 'A'/'B'，由比分判定
    format_weight      REAL    NOT NULL,            -- 录入时快照
    score_multiplier   REAL    NOT NULL,            -- 录入时快照
    note               TEXT,
    created_at         TEXT    NOT NULL DEFAULT (datetime('now'))
);

-- 各模式当前积分缓存（重算重建），主键 (player_id, mode)
CREATE TABLE player_ratings (
    player_id      INTEGER NOT NULL REFERENCES players(id),
    mode           TEXT    NOT NULL,                -- 'singles'/'doubles'
    current_rating REAL    NOT NULL DEFAULT 1500,
    total_matches  INTEGER NOT NULL DEFAULT 0,
    wins           INTEGER NOT NULL DEFAULT 0,
    losses         INTEGER NOT NULL DEFAULT 0,
    updated_at     TEXT    NOT NULL DEFAULT (datetime('now')),
    PRIMARY KEY (player_id, mode)
);

-- 逐场积分变化（重算重建）
CREATE TABLE rating_history (
    id             INTEGER PRIMARY KEY AUTOINCREMENT,
    match_id       INTEGER NOT NULL REFERENCES matches(id) ON DELETE CASCADE,
    mode           TEXT    NOT NULL,
    player_id      INTEGER NOT NULL REFERENCES players(id),
    rating_before  REAL    NOT NULL,
    rating_after   REAL    NOT NULL,
    rating_delta   REAL    NOT NULL,
    k_value        REAL    NOT NULL DEFAULT 24,
    expected_score REAL    NOT NULL,
    actual_score   REAL    NOT NULL,                -- 1/0
    created_at     TEXT    NOT NULL DEFAULT (datetime('now'))
);

CREATE INDEX idx_match_mode_date ON matches(mode, date, id);
CREATE INDEX idx_rh_player_mode  ON rating_history(player_id, mode);
```

**录入校验（应用层强制）**
- 单打选 2 人、双打选 4 人，且两两互不相同。
- `team_a_score ≠ team_b_score`，`winner_team` 由较高分一方推导。
- `format_weight` / `score_multiplier` 由赛制和比分自动算出后写入，不接受手填。

## 5. 核心算法伪代码

### 5.1 录入一场比赛
```
function record_match(input):                  # input.mode = singles/doubles
    team_a = [A1] 或 [A1,A2];  team_b = [B1] 或 [B1,B2]
    validate 人数对应 mode、两两不重复、比分不相等
    winner_team = 'A' if team_a_score > team_b_score else 'B'
    weight = FORMAT_WEIGHT[match_type]
    mult   = score_multiplier(match_type, score_a, score_b)
    insert into matches(... mode ...)          # 含 weight, mult 快照
    rebuild(mode)                              # 见 5.4：重算该模式
```

### 5.2 比分系数
```
function score_multiplier(type, sa, sb):
    w, l = max(sa,sb), min(sa,sb); diff = w - l
    tiebreak7:    diff<=2→0.90; ==3→1.00; <=5→1.10; else→1.20
    tiebreak11:   diff<=2→0.90; <=4→1.00; <=6→1.10; else→1.20
    short4:       diff==1→0.90; ==2→1.00; ==3→1.12; else→1.20
    standard_set: 查表(w,l)，未命中按 diff 兜底
```

### 5.3 状态标签
```
function status_label(total_in_mode, days_since_last_in_mode):
    if days_since_last != null and days_since_last > 60: return "非活跃"
    if total >= 10: return "稳定"
    if total >= 4 : return "观察中"
    if total >= 1 : return "样本少"
    return "新登记"
```

### 5.4 全量重算（某模式，录入/删除后必跑）
```
function rebuild(mode):
    ratings = {每位球员: 1500}
    清空该 mode 的 player_ratings 与 rating_history
    for match in (该 mode 全部比赛 order by date, id):   # 严格按时间
        deltas = compute_match_deltas(ratings, match)
        断言 sum(deltas) ≈ 0                              # 零和自检
        写 rating_history，累加 player_ratings，更新 ratings
```
> K 固定、零和、纯函数，重算结果确定可复现。公司内部量级（几千场内）性能无忧。

## 6. Python 计算函数示例（单打 / 双打通用）

```python
import math

K = 24
INITIAL_RATING = 1500.0
FORMAT_WEIGHT = {"tiebreak7": 0.45, "tiebreak11": 0.60,
                 "short4": 0.80, "standard_set": 1.00}


def expected_score(rating_team_avg: float, rating_opp_avg: float) -> float:
    return 1.0 / (1.0 + 10 ** ((rating_opp_avg - rating_team_avg) / 400.0))


def score_multiplier(match_type: str, score_a: int, score_b: int) -> float:
    w, l = max(score_a, score_b), min(score_a, score_b)
    diff = w - l
    if match_type == "tiebreak7":
        if diff <= 2: return 0.90
        if diff == 3: return 1.00
        if diff <= 5: return 1.10
        return 1.20
    if match_type == "tiebreak11":
        if diff <= 2: return 0.90
        if diff <= 4: return 1.00
        if diff <= 6: return 1.10
        return 1.20
    if match_type == "short4":
        return {1: 0.90, 2: 1.00, 3: 1.12}.get(diff, 1.20)
    if match_type == "standard_set":
        table = {(7, 6): 0.85, (7, 5): 0.95, (6, 4): 1.00, (6, 3): 1.08,
                 (6, 2): 1.15, (6, 1): 1.20, (6, 0): 1.25}
        if (w, l) in table:
            return table[(w, l)]
        return {1: 0.85, 2: 0.95, 3: 1.08, 4: 1.15, 5: 1.20}.get(diff, 1.25)
    raise ValueError(f"未知赛制: {match_type}")


def _team_avg(ratings, ids):
    return sum(ratings[i] for i in ids) / len(ids)


def compute_match_deltas(ratings: dict, match: dict) -> dict:
    """
    match: {team_a:[id,...], team_b:[id,...], score_a, score_b, match_type}
           team 长度 1（单打）或 2（双打），两端一致。
    返回: {player_id: {before, expected, actual, delta, after}}
    """
    a, b = match["team_a"], match["team_b"]
    r_a, r_b = _team_avg(ratings, a), _team_avg(ratings, b)
    e_a = expected_score(r_a, r_b)
    e_b = 1.0 - e_a
    weight = FORMAT_WEIGHT[match["match_type"]]
    mult = score_multiplier(match["match_type"], match["score_a"], match["score_b"])
    factor = K * weight * mult
    a_win = match["score_a"] > match["score_b"]

    out = {}
    for pid in a:
        s = 1.0 if a_win else 0.0
        delta = factor * (s - e_a)
        out[pid] = {"before": ratings[pid], "expected": round(e_a, 4),
                    "actual": s, "delta": delta, "after": ratings[pid] + delta}
    for pid in b:
        s = 0.0 if a_win else 1.0
        delta = factor * (s - e_b)
        out[pid] = {"before": ratings[pid], "expected": round(e_b, 4),
                    "actual": s, "delta": delta, "after": ratings[pid] + delta}
    return out


def assert_zero_sum(deltas: dict, tol: float = 1e-9) -> None:
    total = sum(d["delta"] for d in deltas.values())
    assert abs(total) < tol, f"非零和: {total}"


def status_label(total_matches: int, days_since_last):
    if days_since_last is not None and days_since_last > 60:
        return "非活跃"
    if total_matches >= 10: return "稳定"
    if total_matches >= 4:  return "观察中"
    if total_matches >= 1:  return "样本少"
    return "新登记"


def rebuild(matches: list, player_ids: list, mode: str):
    """按时间重放某模式全部比赛，返回最终 ratings 及逐场 history。"""
    ratings = {pid: INITIAL_RATING for pid in player_ids}
    history = []
    rows = sorted((m for m in matches if m["mode"] == mode),
                  key=lambda x: (x["date"], x["id"]))
    for m in rows:
        deltas = compute_match_deltas(ratings, m)
        assert_zero_sum(deltas)
        for pid, d in deltas.items():
            ratings[pid] = d["after"]
            history.append({"match_id": m["id"], "mode": mode, "player_id": pid, **d})
    return ratings, history
```

> 显示精度：内部 float 全精度保证零和闭合；UI 上 Elo 取整、单场 Δ 取 1 位小数。

## 7. 后续可扩展功能（第二阶段及以后）

- **球员详情 / 对局历史 / 规则页**：第二阶段的三块主功能。
- **不确定度模型**：升级 Glicko-2 或给积分加置信区间，比单 K 更能区分「打得多的稳定分」与「打得少的浮动分」。
- **搭档默契**：统计固定组合实际 vs 理论胜率（仅展示，不进核心积分）。
- **赛季机制**：每季软重置（向 1500 回归 X%）。
- **混双 / 跨模式总榜**：如需把单双打合成一个总分再讨论加权方式。
- **成就**：连胜榜、月度 MVP、逆袭榜（低分赢高分）。
- **导出与通知**：排行榜导出 Excel；企业群定时推送周榜。

## 8. 技术实现

### 8.1 当前已落地架构

第一阶段当前实现已经不是“单文件前端自带共享存储”，而是：

- 前端：手机优先的单页 HTML
- 后端：Node.js HTTP 服务
- 数据库：SQLite（`tennis.db`）
- 鉴权：基于 `TENNIS_ADMIN_PASSWORD` 的录入权限控制

当前访问行为：

- 排行榜与球员列表可直接查看
- 新增比赛、删除比赛、维护球员需要先输入密码
- 所有访问者共享同一份 SQLite 数据

### 8.2 当前实现细节

- 真值仍然是 `players` + `matches`
- `player_ratings` 与 `rating_history` 仍然是重算产物
- 每次新增/删除比赛后按模式全量重算，保证 Elo 路径一致
- 首页 HTML 会内嵌首屏 `players` / `matches` 数据，减少首次打开时的额外接口往返
- 页面已移除外部字体依赖，改用系统字体，以提升手机端和公网访问稳定性
- 排行榜背景主题和卡片样式集中在 CSS theme class 中，业务渲染仅输出排名数据
- `layout_test.html` / `/layout-test` 作为独立图文排版试验页，不读取或修改真实业务数据

### 8.3 部署方式

当前仓库已经支持：

- 本地 Node 直接启动
- Docker / Docker Compose 部署
- SQLite 数据目录挂载持久化
- 运行中数据库在线备份脚本

推荐部署形态：

- 小团队 / 内网 / 轻量公网：Node.js + SQLite + Docker
- 单机长期运行：定时执行数据库备份脚本，保留最近 N 天备份

### 8.4 后续演进方向

如果后续访问人数、并发或分析需求显著增长，可以继续沿用本文档的积分规则与表结构，向下列方向演进：

- 前端：Vue / React
- 后端：FastAPI / Node API
- 数据库：PostgreSQL

无论后端框架如何变化，核心原则不变：

- `players` 与 `matches` 是真值
- 积分逻辑保持固定
- 排行、历史、详情等展示都基于可重放的比赛数据生成
